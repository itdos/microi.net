# データソースエンジン

## 前書き
> * データソースエンジンの概念は主にデータソースを提供するために、その本質はインターフェースエンジンと同じで、Sqlデータソース、普通のデータソースの2つのモードが追加されました。
> * データソースエンジンの【タイプ】が【v 8データソース】の場合、この時は【インターフェースエンジン】と何の違いもありません。

## Sqlデータソース
> * 直接コードエディターにsql文を入力し、 $ current user.* $ 変数をサポートしています
> * メリットは、インタフェースエンジンのようにV8.Db.FromSql() を使用してsql文を実行するのではなく、sql文を直接記述することで、一部の実装エンジニアはこのパターンが好きです
> * サンプルコード:
```sql
SELECT * FROM tableName WHERE UserId = '$CurrentUser.Id$'
```

## 一般データソース
> * コードエディタで直接jsonデータを入力します
> * メリットは究極のパフォーマンスで、v 8エンジンの基礎を呼び出す必要はなく、いくつかの基礎データのストレージ: 全国の省市区住所などのデータを直接返すことができます
> * サンプルコード:
```json
[{ Id : 1, Name : '张三' }, { Id : 2, Name : '李四' }]
```
```json
{ Id : 1, Name : '张三' }
```
