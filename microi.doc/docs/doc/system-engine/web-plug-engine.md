# 数据源引擎

## 前言
>* 数据源引擎的概念为主要提供数据源，其本质与接口引擎差不多，额外增加了Sql数据源、普通数据源2种模式
>* 当数据源引擎的【类型】为【V8数据源】时，此时与【接口引擎】无任何区别

## Sql数据源
>* 直接在代码编辑器中输入sql语句，支持$CurrentUser.*$变量
>* 优势是无需像接口引擎一样使用V8.Db.FromSql()去执行sql语句，而是直接编写sql语句，一些实施工程师比较喜欢这种模式
>* 示例代码：
```sql
SELECT * FROM tableName WHERE UserId = '$CurrentUser.Id$'
```

## 普通数据源
>* 直接在代码编辑器中输入json数据
>* 优势是极致的性能，无需调用V8引擎底层，直接返回数据，如一些基础数据的存储：全国省市区地址等数据
>* 示例代码：
```json
[{ Id : 1, Name : '张三' }, { Id : 2, Name : '李四' }]
```
```json
{ Id : 1, Name : '张三' }
```

## V8数据源
>* 此时与接口引擎无任何区别
